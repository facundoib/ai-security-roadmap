# Contributing

Thanks for considering a contribution. This roadmap is intentionally opinionated and centrally curated — please read this before opening anything.

## What is accepted

**Issues**, via the templates:

- [Suggest a resource](.github/ISSUE_TEMPLATE/suggest-resource.yml) — propose a book, paper, video, blog post, doc, lab, or podcast for a specific week.
- [Typo, broken link, or factual error](.github/ISSUE_TEMPLATE/typo-or-broken-link.yml) — report a fix.

For anything that does not fit a template, email [facundoibarzabal@gmail.com](mailto:facundoibarzabal@gmail.com) instead of opening a free-form issue.

## What is not accepted

**Pull requests against content** (`src/data/weeks.ts`, page copy, week structure). The curation is the value of this project. Open an issue and the maintainer will decide.

PRs against content will be closed with a link back to this file. PRs that fix infrastructure (build config, accessibility, dependency upgrades) are evaluated case-by-case.

## How suggestions are evaluated

A new resource has to either:

- close a real gap that no existing resource for that week covers, or
- replace something already there that it clearly beats.

"It is also good" is not enough. The roadmap is short on purpose.
